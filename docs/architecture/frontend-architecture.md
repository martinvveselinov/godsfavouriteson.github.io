# Frontend Architecture — stoyan-game

Binding rules for all frontend work. Referenced by frontend, test, PR review, and documentation commands. Violations are PR blockers.

## Stack

- Vanilla
- UI Library: none
- Language: TypeScript
- Dev server: `npm run dev`

---

## Project Structure

```
frontend/src/
├── views/          # Route-level pages
├── components/     # Reusable UI components
├── stores/         # State management
├── api/            # HTTP client wrappers
├── composables/    # Shared composition functions
├── router/         # Route definitions + guards
├── styles/         # Global styles
├── locales/        # i18n translation files
└── types/          # Shared TypeScript interfaces
```

---

## Component Rules

- All components use the project's canonical pattern.
- Props and emits must be typed.
- No hardcoded strings in templates — use i18n keys.

## State Management

- Stores never call the backend directly — always through api/ functions.
- State, getters, and actions must be typed.

## API Layer

- All HTTP calls go through the shared HTTP client.
- Never use raw fetch/axios outside the API layer.
- Every API function must have explicit request/response interfaces.
- Handle 401 → refresh/redirect; 403 → show error.

## Design System

- Use the UI library's components for all layout, forms, tables, dialogs.
- No custom CSS that duplicates library utilities.
- Reusable primitives in components/common/ — reuse before writing new ones.
